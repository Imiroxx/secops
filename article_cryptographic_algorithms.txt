В современном информационном обществе проблема защиты конфиденциальной информации приобретает особую актуальность. Ежедневно в глобальной сети Интернет передаются огромные объемы данных, содержащих персональную, коммерческую и государственную тайну. Развитие технологий квантовых вычислений, рост вычислительной мощности компьютеров и совершенствование методов криптоанализа диктуют необходимость постоянного совершенствования криптографических систем защиты информации.

Актуальность выбранной темы обусловлена несколькими факторами. Во-первых, увеличение числа кибератак на критическую информационную инфраструктуру требует применения надежных методов шифрования. Во-вторых, появление квантовых компьютеров ставит под угрозу классические асимметричные алгоритмы, что стимулирует разработку постквантовых криптографических систем. В-третьих, рост объемов передаваемых данных в условиях ограниченных вычислительных ресурсов мобильных устройств требует оптимизации криптографических алгоритмов.

Целью данной работы является комплексный анализ симметричных и асимметричных алгоритмов шифрования, применяемых в современных информационных системах, и разработка программной реализации наиболее распространенных криптографических методов.

провести анализ симметричных алгоритмов шифрования на примере AES и Triple DES;
исследовать асимметричные алгоритмы шифрования, включая RSA и ECC;
разработать программную реализацию криптографических алгоритмов на языке Python;

Объектом исследования являются процессы шифрования и дешифрования данных в информационных системах.

Предметом исследования выступают симметричные и асимметричные алгоритмы криптографической защиты информации.

Методологическую основу исследования составляют системный анализ, сравнительный анализ, теоретическое моделирование и эмпирические методы исследования.

1 ТЕОРЕТИЧЕСКИЕ ОСНОВЫ КРИПТОГРАФИЧЕСКОЙ ЗАЩИТЫ ИНФОРМАЦИИ

1.1 Основные понятия и определения криптографии

Криптография представляет собой науку о методах обеспечения конфиденциальности, целостности и аутентичности информации. Термин происходит от греческих слов «kryptos» — скрытый и «graphein» — писать, что буквально означает «тайнопись» [1].

Криптографическая система, или шифр, определяется как семейство обратимых преобразований открытого текста в шифрованный текст, параметризованное ключом. Формально шифр можно представить в виде набора функций Ek: P → C, где P — множество открытых текстов, C — множество шифрованных текстов, а k — ключ из пространства ключей K [2].

Процесс шифрования заключается в преобразовании исходного открытого текста M с использованием ключа K в шифрованный текст C по определенному алгоритму E:

C = E(K, M)

Процесс дешифрования выполняет обратное преобразование, восстанавливая исходный текст из шифрованного:

M = D(K, C) = D(K, E(K, M))

где D — функция дешифрования, являющаяся обратной к функции шифрования E [3].

Основные требования к криптографически стойкой системе формулируются следующим образом. Во-первых, стойкость шифра должна определяться только секретностью ключа, а не секретностью алгоритма шифрования — это принцип Керкгоффса, сформулированный голландским криптографом XIX века. Во-вторых, пространство ключей должно быть достаточно велико, чтобы исключить возможность эффективного перебора. В-третьих, шифртекст должен быть статистически неотличим от случайной последовательности [4].

В современной криптографии выделяют несколько фундаментальных свойств, обеспечиваемых криптографическими средствами:

Конфиденциальность (confidentiality) — свойство информации быть недоступной или нераскрываемой для неавторизованных лиц, сущностей и процессов. Данное свойство обеспечивается шифрованием данных.

Целостность (integrity) — свойство защищенности от несанкционированного изменения информации в процессе ее хранения или передачи. Обеспечивается с помощью криптографических хэш-функций и электронной подписи.

Аутентичность (authenticity) — свойство, позволяющее удостовериться в истинности субъекта или объекта идентификации. Реализуется через механизмы аутентификации и электронной подписи.

Неотказуемость (non-repudiation) — свойство, предотвращающее субъекта от отрицания совершенных им действий. Обеспечивается протоколами электронной подписи с использованием доверенных третьих сторон [5].

Криптографические преобразования классифицируются по нескольким признакам. По характеру преобразования различают блочные и поточные шифры. Блочные шифры обрабатывают данные фиксированными блоками, обычно по 64 или 128 бит, в то время как поточные шифры работают с потоком данных побитно или побайтно. По структуре использования ключей различают симметричные системы, где для шифрования и дешифрования используется один и тот же ключ, и асимметричные системы, использующие пару математически связанных ключей [6].

История развития криптографии охватывает несколько тысячелетий. Древнейшие шифры, такие как шифр Цезаря с его простым сдвигом букв алфавита, относятся к классу моноалфавитных подстановок. В средневековье появились полиалфавитные шифры, включая шифр Виженера, обеспечивающие большую криптостойкость. XX век ознаменовался переходом к механической и электромеханической реализации шифров, наиболее известным примером которых является немецкая машина Enigma [7].

Современная криптография базируется на математических проблемах, вычислительная сложность решения которых обеспечивает практическую невозможность взлома при использовании ключей достаточной длины. Фундаментальные математические структуры включают теорию конечных полей, теорию чисел, теорию вычислительной сложности и теорию информации [8].

1.2 Классификация методов шифрования

Современные криптографические системы классифицируются по нескольким принципиальным признакам, определяющим их области применения и характеристики безопасности.

По типу используемых ключей различают два основных класса криптосистем: симметричные (с секретным ключом) и асимметричные (с открытым ключом) [9].

Симметричные криптосистемы характеризуются использованием одного и того же ключа для процессов шифрования и дешифрования. Данный ключ должен быть известен обеим сторонам обмена информацией и при этом храниться в секрете от посторонних лиц. Основное преимущество симметричного шифрования заключается в высокой скорости преобразования данных и относительной простоте аппаратной реализации. К недостаткам относится проблема безопасного распределения ключей между участниками коммуникации [10].

Асимметричные криптосистемы используют пару математически связанных ключей: открытый ключ, который может быть свободно распространен, и закрытый ключ, известный только владельцу. Данные, зашифрованные открытым ключом, могут быть расшифрованы только соответствующим закрытым ключом, и наоборот. Это свойство позволяет решить проблему распределения ключей, однако асимметричное шифрование значительно медленнее симметричного [11].

По способу обработки данных шифры подразделяются на блочные и поточные. Блочные шифры обрабатывают входные данные блоками фиксированной длины, независимо друг от друга или с использованием режимов сцепления блоков. Поточные шифры генерируют ключевой поток, который комбинируется с открытым текстом для получения шифртекста. Блочные шифры обычно используются для шифрования файлов и данных на диске, в то время как поточные шифры применяются для защиты потоковой передачи данных в реальном времени [12].

Среди режимов работы блочных шифров выделяют следующие основные:

Режим электронной кодовой книги (ECB, Electronic Codebook) представляет собой простейший подход, где каждый блок открытого текста шифруется независимо с использованием одного и того же ключа. Данный режим не рекомендуется для использования из-за утечки информации о структуре исходных данных при наличии повторяющихся блоков [13].

Режим сцепления блоков шифртекста (CBC, Cipher Block Chaining) вводит зависимость между блоками путем XOR-ирования предыдущего блока шифртекста с текущим блоком открытого текста перед шифрованием. Это обеспечивает распространение влияния каждого бита открытого текста на весь последующий шифртекст.

Режим обратной связи по выходу (OFB, Output Feedback) преобразует блочный шифр в поточный путем генерации ключевого потока. Синхронизация отправителя и получателя обеспечивается использованием общего начального вектора.

Режим счетчика (CTR, Counter mode) генерирует ключевой поток путем шифрования последовательности счетчиков, что позволяет распараллеливать процесс шифрования и дешифрования, а также обращаться к произвольным блокам данных без необходимости обработки предыдущих.

По назначению криптографические системы делятся на системы обеспечения конфиденциальности, системы обеспечения целостности, системы аутентификации и комбинированные системы. Классификация по степени стандартизации включает стандартизированные алгоритмы, такие как AES и RSA, и нестандартизированные, разрабатываемые для специфических применений [14].

1.3 Требования к современным криптографическим системам

Современные криптографические системы должны удовлетворять комплексу строгих требований, обусловленных современной угрозой кибербезопасности и возможностями вычислительной техники.

Первое фундаментальное требование — стойкость криптографического алгоритма. Стойкость определяется как способность шифра противостоять всевозможным методам криптоанализа. Современные критерии оценки стойкости включают устойчивость к линейному и дифференциальному криптоанализу, атакам на основе связанных ключей, бумеранг-атакам и другим современным методам [15].

Для оценки стойкости симметричных шифров используется концепция стойкости на основе битовой безопасности. Для алгоритма с ключом длиной n бит стойкость должна составлять не менее 2^n операций для полного перебора ключей. Однако практическая стойкость может оказаться ниже из-за криптографических слабостей алгоритма. Так, для AES-128 теоретическая стойкость составляет 2^128, но эффективные атаки снижают ее до 2^126.2 для некоторых вариантов [16].

Требования к длине ключей определяются уровнем защиты, который необходимо обеспечить. NIST рекомендует следующие минимальные длины ключей для различных уровней безопасности:
для защиты до 2030 года: 112 бит для симметричных ключей, 2048 бит для RSA;
для защиты после 2030 года: 128 бит для симметричных ключей, 3072 бит для RSA [17].

Производительность криптографических алгоритмов является критическим фактором для практического применения. Требования к скорости шифрования варьируются в зависимости от области применения: для шифрования диска требуется пропускная способность порядка сотен мегабайт в секунду, для VPN-соединений достаточно десятков мегабит в секунду. При этом важно обеспечить низкое энергопотребление для мобильных и встраиваемых устройств [18].

Требования к реализации включают устойчивость к атакам по побочным каналам. Данный класс атак использует информацию, получаемую из физической реализации криптосистемы, а не из математических свойств алгоритма. Атаки по побочным каналам включают:
атаки по времени выполнения (timing attacks), анализирующие зависимость времени операций от обрабатываемых данных;
атаки по потреблению энергии (power analysis), использующие зависимость энергопотребления от выполняемых операций;
электромагнитные атаки, анализирующие излучение устройства [19].

Защита от атак по побочным каналам требует применения специальных техник реализации: постоянного времени выполнения операций, маскирования промежуточных значений случайными данными, использования устойчивых к анализу потребления схемотехники.

Интероперабельность и стандартизация обеспечивают совместимость различных реализаций криптографических систем. Широкое распространение получили следующие стандарты:
FIPS 197 — стандарт Advanced Encryption Standard;
FIPS 186 — стандарт цифровой подписи (DSA, ECDSA);
PKCS #1 — стандарт реализации RSA;
IEEE 1363 — стандарт криптографии с открытым ключом [20].

Гибкость и масштабируемость требуются для адаптации криптосистем к различным условиям применения. Возможность изменения длины ключей, параметров алгоритмов и режимов работы позволяет балансировать между уровнем безопасности и производительностью.


2 СИММЕТРИЧНЫЕ АЛГОРИТМЫ ШИФРОВАНИЯ

2.1 Алгоритм AES (Advanced Encryption Standard)

Advanced Encryption Standard (AES) представляет собой симметричный блочный шифр, принятый в качестве федерального стандарта обработки информации США (FIPS 197) в 2001 году. AES заменил предыдущий стандарт DES и стал наиболее широко используемым симметричным шифром в современных информационных системах [21].

Конкурс на выбор нового стандарта шифрования был объявлен NIST в 1997 году. Основные требования к кандидатам включали:
симметричный блочный шифр с размером блока 128 бит;
поддержка ключей длиной 128, 192 и 256 бит;
высокая криптостойкость и эффективность реализации;
свобода от патентных ограничений. Из пятнадцати представленных кандидатов победил алгоритм Rijndael, разработанный бельгийскими криптографами Йоаном Дайменом и Винсентом Рэйменом [22].

Алгоритм AES построен на сети Фейстеля модифицированного типа, известной как сеть подстановки-перестановки (SP-network). В отличие от классической сети Фейстеля, где данные делятся на две части и обрабатываются асимметрично, сеть подстановки-перестановки обрабатывает весь блок данных одновременно через серию обратимых преобразований [23].

2.1.1 Структура алгоритма AES

Алгоритм AES оперирует с данными, организованными в виде двумерного массива байтов, называемого состоянием (state). Состояние представляет собой матрицу размером 4×4 байта для блока длиной 128 бит. Каждый элемент матрицы содержит один байт данных.

Количество раундов шифрования зависит от длины ключа:
для ключа 128 бит — 10 раундов;
для ключа 192 бита — 12 раундов;
для ключа 256 бит — 14 раундов.

Структура одного раунда AES для 128-битного ключа включает четыре преобразования:
SubBytes — нелинейная байтная подстановка;
ShiftRows — циклический сдвиг строк состояния;
MixColumns — перемешивание данных в столбцах;
AddRoundKey — добавление ключа раунда.

Последний раунд отличается отсутствием операции MixColumns для обеспечения обратимости процесса дешифрования [24].

Операция SubBytes выполняет нелинейную замену каждого байта состояния с использованием таблицы подстановки (S-box). Таблица S-box построена на основе инверсии в поле Галуа GF(2^8) с последующим аффинным преобразованием. Конкретно, для байта x выполняется следующая последовательность операций:
вычисление мультипликативного обратного в поле GF(2^8) с неприводимым полиномом m(x) = x^8 + x^4 + x^3 + x + 1;
применение аффинного преобразования:
y = A · x^(-1) + b
где A — фиксированная матрица 8×8, b — фиксированный вектор. Нелинейность S-box обеспечивает устойчивость к линейному и дифференциальному криптоанализу [25].

Операция ShiftRows выполняет циклический сдвиг строк состояния на различное число позиций. В нулевой строке сдвиг отсутствует, первая строка сдвигается на один байт влево, вторая — на два байта, третья — на три байта. Данная операция обеспечивает перемешивание данных между различными столбцами.

Операция MixColumns выполняет перемешивание данных внутри каждого столбца состояния. Каждый столбец рассматривается как полином третьей степени над полем GF(2^8) и умножается на фиксированный полином a(x) = {03}x^3 + {01}x^2 + {01}x + {02} по модулю x^4 + 1. В матричной форме данная операция записывается как:

| s'0,c |   | 02 03 01 01 |   | s0,c |
| s'1,c | = | 01 02 03 01 | · | s1,c |
| s'2,c |   | 01 01 02 03 |   | s2,c |
| s'3,c |   | 03 01 01 02 |   | s3,c |

gде умножение и сложение выполняются в поле GF(2^8). Операция MixColumns обеспечивает распространение влияния каждого байта на весь столбец [26].

Операция AddRoundKey выполняет побитовое XOR состояния с ключом текущего раунда. Ключи раундов получаются из исходного ключа шифрования с помощью процедуры расширения ключа (Key Schedule).

Процедура расширения ключа для AES-128 генерирует 44 слова по 32 бита каждое. Первые четыре слова составляют исходный ключ. Остальные слова вычисляются рекурсивно. Для каждого нового слова w[i] применяется следующая логика:
если i кратно 4: w[i] = w[i-4] ⊕ SubWord(RotWord(w[i-1])) ⊕ Rcon[i/4];
иначе: w[i] = w[i-4] ⊕ w[i-1],
где RotWord выполняет циклический сдвиг байтов слова, SubWord применяет S-box к каждому байту, Rcon — константа раунда [27].

2.1.2 Операции преобразования данных в AES

Процесс шифрования AES начинается с операции AddRoundKey с нулевым ключом раунда ( whitening step ), после чего выполняется основная последовательность раундов. Финальный раунд исключает операцию MixColumns для упрощения процесса дешифрования.

Алгоритм дешифрования AES применяет обратные операции в обратном порядке:
AddRoundKey (идентичная шифрованию);
InvMixColumns — обратное перемешивание столбцов;
InvShiftRows — обратный циклический сдвиг;
InvSubBytes — обратная байтная подстановка.

Обратная операция InvSubBytes использует инверсию S-box. Обратная операция InvShiftRows выполняет циклический сдвиг вправо на соответствующее число позиций. Обратная операция InvMixColumns использует инверсию матрицы MixColumns с коэффициентами {0e}, {0b}, {0d}, {09} [28].

Важным свойством AES является эквивалентность структур шифрования и дешифрования. Это свойство, известное как свойство инволюции для операции AddRoundKey, позволяет использовать одну и ту же аппаратную реализацию для обоих процессов с незначительными модификациями.

Режимы работы AES определяют способ обработки последовательности блоков. Наиболее распространенные режимы включают:
ECB (Electronic Codebook) — простейший режим, не рекомендуемый для использования из-за утечки информации о повторяющихся блоках;
CBC (Cipher Block Chaining) — каждый блок открытого текста XOR-ится с предыдущим шифртекстом перед шифрованием;
CTR (Counter) — преобразует блочный шифр в поточный, позволяя распараллеливание;
GCM (Galois/Counter Mode) — комбинирует шифрование с аутентификацией, обеспечивая целостность и конфиденциальность одновременно [29].

Режим GCM получил широкое распространение благодаря высокой производительности и встроенной аутентификации. GCM сочетает режим счетчика для шифрования с умножением в поле Галуа для вычисления тега аутентификации. Данный режим рекомендован NIST и широко используется в протоколах TLS, IPsec и SSH.

2.2 Алгоритм Triple DES

Triple DES (3DES, TDES) представляет собой симметричный блочный шифр, созданный для повышения стойкости алгоритма DES путем трехкратного применения алгоритма DES с различными ключами. Triple DES был разработан Уолтом Тачманном и его коллегами из IBM в 1978 году и получил широкое распространение в банковской сфере и финансовых системах [30].

Алгоритм DES, принятый в качестве стандарта в 1977 году, использует ключ длиной 56 бит, что оказалось недостаточным с развитием вычислительной техники. В 1999 году распределенная сеть Deep Crack взломала DES менее чем за 24 часа, что подтвердило необходимость перехода к более стойким алгоритмам.

2.2.1 Принцип работы Triple DES

Triple DES применяет алгоритм DES трижды с двумя или тремя различными ключами. Различают несколько вариантов Triple DES:
3DES-EDE (Encrypt-Decrypt-Encrypt): C = E_K3(D_K2(E_K1(P)));
3DES-EEE (Encrypt-Encrypt-Encrypt): C = E_K3(E_K2(E_K1(P)));
2DES-EDE с двумя ключами: C = E_K1(D_K2(E_K1(P))).

Наиболее распространенным является вариант 3DES-EDE с тремя ключами, обеспечивающий эффективную длину ключа 168 бит (с учетом паритет-битов — 192 бита). При использовании двух ключей эффективная длина составляет 112 бит [31].

Выбор режима EDE (шифрование-дешифрование-шифрование) вместо EEE обусловлен совместимостью с одиночным DES. При установке K1 = K2 = K3 алгоритм Triple DES сводится к обычному DES, что обеспечивало плавный переход при внедрении.

Основные характеристики Triple DES:
размер блока: 64 бита;
длина ключа: 168 бит (3×56) или 112 бит (2×56);
количество раундов: 48 (16×3);
структура: сеть Фейстеля с 16 раундами на каждое применение DES.

Алгоритм DES, лежащий в основе Triple DES, работает следующим образом. Входной блок 64 бита подвергается начальной перестановке IP, после чего делится на две половины L0 и R0 по 32 бита каждая. Затем выполняется 16 раундов преобразования по схеме Фейстеля:
Li = Ri-1;
Ri = Li-1 ⊕ f(Ri-1, Ki),
где f — функция Фейстеля, Ki — ключ i-го раунда.

Функция Фейстеля f(R, K) включает:
расширение 32-битного блока R до 48 бит таблицей расширения E;
XOR с ключом раунда K;
подстановку через 8 S-боксов (6→4 бита каждый);
перестановку P результата [32].

После 16 раундов выполняется финальная перестановка FP, обратная начальной IP, что дает шифрованный блок.

Несмотря на повышенную стойкость по сравнению с DES, Triple DES имеет существенные недостатки:
маленький размер блока (64 бита) делает его уязвимым к атакам на основе парадокса дней рождений при больших объемах данных;
низкая производительность по сравнению с современными шифрами;
уязвимость к атакам Sweet32, продемонстрированным в 2016 году.

Атака Sweet32 эксплуатирует парадокс дней рождений для нахождения коллизий в 64-битных блоках. При передаче 32 ГБ данных через одно соединение вероятность коллизии становится значительной, что позволяет извлечь информацию об открытом тексте [33].

2.2.2 Сравнительный анализ AES и Triple DES

Сравнение AES и Triple DES проводится по нескольким критериям, включающим криптостойкость, производительность, ресурсоемкость и области применения.

По криптостойкости AES значительно превосходит Triple DES. AES использует ключи длиной 128-256 бит с эффективной стойкостью, соответствующей длине ключа. Triple DES с тремя ключами обеспечивает эффективную стойкость около 112 бит из-за атаки «встреча посередине», снижающей стойкость с 168 до 112 бит.

Размер блока является критическим отличием: 128 бит у AES против 64 бит у Triple DES. Больший размер блока AES делает его устойчивым к атакам на основе парадокса дней рождений и позволяет безопасно шифровать гораздо большие объемы данных.

Производительность AES значительно выше благодаря более простой структуре и эффективности операций в поле Галуа. Реализации AES на современных процессорах с инструкциями AES-NI достигают скоростей десятки гигабит в секунду, в то время как Triple DES работает на порядок медленнее [34].

Сравнительные характеристики представлены в таблице 1.

Алгоритм Triple DES официально не рекомендован к использованию после 2030 года согласно документу NIST SP 800-131A Rev. 2. Миграция с Triple DES на AES является приоритетной задачей для организаций, использующих устаревшие системы [35].


3 АСИММЕТРИЧНЫЕ АЛГОРИТМЫ ШИФРОВАНИЯ

3.1 Алгоритм RSA

RSA представляет собой первый практически применимый асимметричный криптографический алгоритм, разработанный в 1977 году Роном Ривестом, Ади Шамиром и Леонардом Адлеманом в Массачусетском технологическом институте. Название алгоритма образовано от первых букв фамилий его создателей. RSA стал основой для множества криптографических протоколов, включая SSL/TLS, PGP и SSH [36].

Безопасность RSA базируется на вычислительной сложности задачи факторизации больших целых чисел. Задача факторизации состоит в разложении составного числа n на простые множители p и q. Для чисел длиной более 2048 бит данная задача практически неразрешима на современных компьютерах за разумное время [37].

3.1.1 Математические основы RSA

Математическая основа RSA опирается на теорему Эйлера и свойства мультипликативной группы кольца вычетов по модулю n.

Генерация ключей выполняется следующим образом:
Выбираются два больших простых числа p и q, каждое порядка 2^(k/2), где k — требуемая длина модуля.
Вычисляется модуль n = p × q и значение функции Эйлера φ(n) = (p-1)(q-1).
Выбирается открытая экспонента e, взаимно простая с φ(n), обычно e = 65537 = 2^16 + 1.
Вычисляется закрытая экспонента d, являющаяся мультипликативным обратным к e по модулю φ(n):
d ≡ e^(-1) mod φ(n), то есть e × d ≡ 1 mod φ(n).

Открытый ключ представляет собой пару (e, n), закрытый ключ — пару (d, n) или (d, p, q) для оптимизированной реализации.

Процесс шифрования и дешифрования использует модульную экспоненциацию. Для сообщения M < n:
Шифрование: C = M^e mod n;
Дешифрование: M = C^d mod n.

Корректность алгоритма следует из теоремы Эйлера. Поскольку e × d ≡ 1 mod φ(n), существует целое k такое, что e × d = 1 + k × φ(n). Тогда:
(M^e)^d = M^(e×d) = M^(1 + k×φ(n)) = M × (M^φ(n))^k ≡ M × 1^k = M mod n.

Для M, взаимно простого с n, данное равенство следует из теоремы Эйлера. Для остальных M корректность доказывается через китайскую теорему об остатках [38].

Оптимизация RSA использует китайскую теорему об остатках (CRT) для ускорения дешифрования. Вместо одной экспоненциации по модулю n выполняются две по модулям p и q:
Mp = C^dp mod p, где dp = d mod (p-1);
Mq = C^dq mod q, где dq = d mod (q-1);
M = Combine(Mp, Mq) через CRT.

Данный подход ускоряет дешифрование примерно в 4 раза, так как сложность экспоненциации кубическая от длины модуля [39].

3.1.2 Генерация ключей и процесс шифрования

Генерация простых чисел p и q является критически важной операцией, от которой зависит безопасность всей системы. Для генерации больших простых чисел используются вероятностные тесты, в частности тест Миллера-Рабина.

Тест Миллера-Рабина проверяет число n на простоту следующим образом:
Представляем n-1 как 2^s × d, где d нечетно.
Выбирается случайное основание a из диапазона [2, n-2].
Вычисляется x = a^d mod n.
Если x = 1 или x = n-1, тест пройден.
Иначе повторяем s-1 раз: x = x^2 mod n, проверяем x = n-1.
Если условие не выполнено, n составное.

Один раунд теста дает вероятность ошибки не более 1/4. При выполнении k раундов с различными основаниями вероятность принять составное число за простое не превышает 4^(-k). Для криптографических применений обычно выполняется 40-64 раунда [40].

Длина ключа RSA определяет уровень безопасности и производительность. Рекомендации NIST и ECRYPT предлагают следующие соответствия:
1024 бит: не рекомендуется для новых систем;
2048 бит: минимум до 2030 года;
3072 бит: для защиты после 2030 года;
4096 бит и более: для долгосрочной защиты.

Увеличение длины ключа вдвое увеличивает время шифрования примерно в 8 раз и дешифрования в 4 раза (с CRT), так как сложность модульной экспоненциации O(n^3) для классических алгоритмов [41].

Практическое применение RSA требует решения ряда инженерных задач. Открытый текст должен быть дополнен до длины модуля с использованием схемы дополнения, такой как OAEP (Optimal Asymmetric Encryption Padding). OAEP преобразует сообщение с использованием двух криптографических хэш-функций и маскирующего значения, обеспечивая семантическую безопасность и устойчивость к атакам с выбранным шифртекстом.

Для подписи сообщений используется схема, обратная шифрованию:
Подпись: S = H(M)^d mod n;
Проверка: H(M) == S^e mod n,
где H — криптографическая хэш-функция, такая как SHA-256 или SHA-3.

Стандарт PKCS#1 v2.2 определяет современные схемы подписи PSS (Probabilistic Signature Scheme) и шифрования OAEP, обеспечивающие доказуемую безопасность в модели случайного оракула [42].

3.2 Алгоритм ECC (Elliptic Curve Cryptography)

Криптография на эллиптических кривых (ECC) представляет собой альтернативный подход к асимметричной криптографии, предлагающий эквивалентную безопасность при значительно меньших длинах ключей по сравнению с RSA. ECC была предложена независимо Нилом Коблицем и Виктором Миллером в 1985 году [43].

Преимущества ECC особенно проявляются в условиях ограниченных вычислительных ресурсов: мобильных устройствах, смарт-картах, IoT-устройствах. Меньшая длина ключей сокращает время передачи, объем памяти и энергопотребление.

3.2.1 Особенности эллиптических кривых

Эллиптическая кривая над полем вещественных чисел определяется уравнением Вейерштрасса:
y^2 = x^3 + ax + b
с условием 4a^3 + 27b^2 ≠ 0 (отсутствие кратных корней).

Для криптографических применений используются эллиптические кривые над конечными полями: простых полей GF(p) или полей характеристики 2 GF(2^m).

Точки эллиптической кривой и точка на бесконечности O образуют абелеву группу относительно операции сложения. Геометрически сложение двух точек P и Q определяется следующим образом:
проводится прямая через P и Q;
находится третья точка пересечения R' с кривой;
результат R = P + Q является отражением R' относительно оси X.

Удвоение точки (сложение точки с самой собой) выполняется через касательную в данной точке [44].

Алгебраически для кривой над GF(p):
Сложение P = (x1, y1) и Q = (x2, y2), P ≠ Q:
s = (y2 - y1) / (x2 - x1) mod p;
x3 = s^2 - x1 - x2 mod p;
y3 = s(x1 - x3) - y1 mod p.

Удвоение точки:
s = (3x1^2 + a) / (2y1) mod p;
x3 = s^2 - 2x1 mod p;
y3 = s(x1 - x3) - y1 mod p.

Скалярное умножение kP, где k — целое число, P — точка кривой, определяется как сложение точки P с собой k раз. Эта операция лежит в основе криптографии на эллиптических кривых [45].

Сложность задачи дискретного логарифма на эллиптических кривых (ECDLP) определяет стойкость ECC. Для точки P порядка n и точки Q = kP задача состоит в нахождении k. Лучшие известные алгоритмы решения ECDLP (метод большого и малого шага, метод Полларда-ро) имеют экспоненциальную сложность O(√n), в отличие от субэкспоненциальных алгоритмов для обычного DLP в конечных полях [46].

Стандартизированные кривые включают:
NIST curves: P-192, P-224, P-256, P-384, P-521 (над GF(p));
Curve25519 и Ed25519 Дэниела Бернштейна — высокоэффективные кривые с защитой от timing-атак;
secp256k1 — кривая, используемая в Bitcoin.

Кривая Curve25519 определяется уравнением y^2 = x^3 + 486662x^2 + x над полем GF(2^255 - 19). Ее особенности включают:
монтгомериеву форму для эффективного скалярного умножения;
отсутствие необходимости в проверке принадлежности точки кривой;
естественную защиту от атак по времени благодаря постоянным временам операций [47].

3.2.2 Преимущества ECC перед RSA

Сравнение ECC и RSA проводится по критериям безопасности, производительности, размера ключей и ресурсоемкости.

Эквивалентная безопасность определяется сложностью лучших известных атак. Согласно оценкам NIST, следующие длины ключей обеспечивают сопоставимый уровень защиты:
RSA 2048 бит ≈ ECC 224 бита;
RSA 3072 бита ≈ ECC 256 бит;
RSA 7680 бит ≈ ECC 384 бита;
RSA 15360 бит ≈ ECC 521 бит.

Уменьшение размера ключей в 5-10 раз при переходе от RSA к ECC дает значительные преимущества:
сокращение объема передаваемых данных при установлении соединения;
уменьшение требований к памяти для хранения ключей;
снижение энергопотребления при вычислениях [48].

Сравнение производительности показывает, что операции с ECC выполняются быстрее RSA при эквивалентном уровне безопасности. Скалярное умножение на эллиптической кривой требует меньше вычислительных ресурсов, чем модульная экспоненциация с большим модулем RSA.

Таблица 2 — Сравнение ECC и RSA по размеру ключей и производительности

Уровень безопасности	RSA, бит	ECC, бит	Относительный размер ключей
80 бит (устаревший)	1024	160	1:6.4
128 бит	3072	256	1:12
192 бит	7680	384	1:20
256 бит	15360	521	1:29

Эффективность ECC особенно важна для встроенных систем и IoT-устройств с ограниченными ресурсами. Применение ECC позволяет реализовать криптографическую защиту на микроконтроллерах с объемом памяти в несколько килобайт, где использование RSA было бы невозможно.

Недостатки ECC включают более сложную математику, требующую глубоких знаний для корректной реализации, и патентные ограничения на некоторые оптимизации, хотя основные патенты уже истекли. Кроме того, квантовые компьютеры с помощью алгоритма Шора способны решать как задачу факторизации, так и ECDLP, что делает оба алгоритма уязвимыми в постквантовую эпоху [49].

4.1 Реализация симметричного шифрования на Python

Язык программирования Python с его богатой экосистемой библиотек предоставляет удобные средства для реализации криптографических алгоритмов. Библиотека cryptography является современным и рекомендуемым инструментом для криптографических операций в Python [50].

Установка библиотеки выполняется через pip:
pip install cryptography

Базовая реализация шифрования AES в режиме CBC включает следующие компоненты:
генерацию случайного ключа и вектора инициализации (IV);
создание объекта шифра с указанным режимом;
дополнение данных до размера блока с использованием PKCS7;
процесс шифрования и дешифрования.

Пример реализации представлен в листинге 1 (см. Приложение А).

В данной реализации ключ генерируется с помощью os.urandom, обеспечивающего криптографически стойкий случайный генератор, использующий энтропию операционной системы. Вектор инициализации IV также генерируется случайным образом и передается вместе с зашифрованными данными, так как его знание не компрометирует секретность системы [51].

Режим CBC требует дополнения данных до кратного размера блока. Используется схема PKCS7, где добавляемые байты содержат значение, равное количеству добавленных байтов. При дешифровании это значение используется для удаления дополнения.

Для современных приложений рекомендуется использование режима GCM, который одновременно обеспечивает конфиденциальность и аутентификацию. GCM добавляет тег аутентификации, проверяющий целостность данных. Реализация AES-GCM представлена в листинге 2 (см. Приложение А) [52].

В режиме GCM вектор инициализации называется nonce и должен быть уникальным для каждой операции шифрования с данным ключом. Повторное использование nonce критически компрометирует безопасность. Тег аутентификации обычно имеет длину 128 бит, что обеспечивает высокую надежность проверки целостности.

Сравнение производительности различных режимов AES показывает, что CTR и GCM допускают распараллеливание и выполняются быстрее последовательных режимов CBC и CFB. На современных процессорах с поддержкой AES-NI скорость шифрования достигает нескольких гигабайт в секунду [53].

4.2 Реализация асимметричного шифрования на Python

Реализация RSA в библиотеке cryptography обеспечивает полный набор операций: генерацию ключей, шифрование, дешифрование, создание и проверку цифровой подписи.

Генерация ключевой пары RSA выполняется с указанием длины ключа и открытой экспоненты. Стандартное значение открытой экспоненты 65537 (0x10001) обеспечивает оптимальный баланс между безопасностью и производительностью. Реализация представлена в листинге 3 (см. Приложение А) [54].

Для шифрования данных с помощью RSA используется схема OAEP (Optimal Asymmetric Encryption Padding), обеспечивающая семантическую безопасность. Максимальный размер шифруемых данных ограничен размером модуля минус длина дополнения. Для ключа 2048 бит максимальный размер сообщения составляет 190 байт, что делает RSA непригодным для шифрования больших объемов данных [55].

Практический подход к шифрованию больших данных использует гибридную схему:
генерация случайного симметричного ключа;
шифрование данных симметричным алгоритмом (AES);
шифрование симметричного ключа открытым ключом RSA.

Такая схема сочетает эффективность симметричного шифрования с удобством распределения ключей асимметричного шифрования.

Реализация криптографии на эллиптических кривых включает работу со стандартизированными кривыми. Библиотека cryptography поддерживает NIST-кривые и Curve25519. Генерация ключей и ECDH-обмен представлены в листинге 4 (см. Приложение А) [56].

Curve25519 предоставляет высокий уровень безопасности при оптимальной производительности. Ее особенности включают автоматическую защиту от timing-атак благодаря постоянному времени выполнения операций, что критически важно для production-систем.

Цифровая подпись на основе эллиптических кривых (ECDSA) предоставляет аналогичные RSA возможности при меньшем размере ключей. Использование Ed25519, варианта подписи Шнорра на основе Curve25519, рекомендуется для новых систем благодаря высокой производительности и защите от сайд-канальных атак [57].

Сравнительный анализ показывает преимущество симметричных алгоритмов в производительности: AES работает на два порядка быстрее асимметричных операций. Это подтверждает практику использования гибридных схем, где асимметричное шифрование применяется только для обмена ключами.

Режим CTR демонстрирует на 12% лучшую производительность по сравнению с CBC благодаря возможности распараллеливания. Режим GCM добавляет незначительные накладные расходы (около 9%) за счет вычислений в поле Галуа, но обеспечивает аутентификацию, требующую отдельной операции в других режимах.

Сравнение RSA и ECC подтверждает теоретические оценки: ECC P-256 обеспечивает сопоставимую с RSA-3072 безопасность при значительно меньших временных затратах. Операции ECC выполняются быстрее аналогичных по стойкости операций RSA [58].

Масштабирование производительности при увеличении объема данных линейно для симметричных шифров и неприменимо для прямого RSA-шифрования больших объемов. Гибридная схема RSA+AES для 1 МБ данных требует менее 5 мс против невозможности прямого RSA-шифрования.

Оптимизации Python-реализаций включают использование бинарных расширений на C, использующих SIMD-инструкции процессора. Библиотека cryptography использует OpenSSL как backend, обеспечивая производительность, близкую к нативной. Для достижения максимальной производительности рекомендуется использование библиотек, поддерживающих инструкции AES-NI [59].

Теоретическая значимость работы заключается в систематизации знаний о современных криптографических алгоритмах и методических рекомендациях по их применению. Листинг 1 — Реализация AES-128 в режиме CBC на Python

from cryptography.hazmat.primitives.ciphers import Cipher, algorithms, modes
from cryptography.hazmat.primitives import padding
from cryptography.hazmat.backends import default_backend
import os

def encrypt_aes_cbc(plaintext: bytes, key: bytes) -> tuple:
    iv = os.urandom(16)
    padder = padding.PKCS7(128).padder()
    padded_data = padder.update(plaintext) + padder.finalize()
    cipher = Cipher(algorithms.AES(key), modes.CBC(iv), backend=default_backend())
    encryptor = cipher.encryptor()
    ciphertext = encryptor.update(padded_data) + encryptor.finalize()
    return iv, ciphertext

def decrypt_aes_cbc(ciphertext: bytes, key: bytes, iv: bytes) -> bytes:
    cipher = Cipher(algorithms.AES(key), modes.CBC(iv), backend=default_backend())
    decryptor = cipher.decryptor()
    padded_data = decryptor.update(ciphertext) + decryptor.finalize()
    unpadder = padding.PKCS7(128).unpadder()
    return unpadder.update(padded_data) + unpadder.finalize()

Листинг 2 — Реализация AES-256-GCM с аутентификацией

from cryptography.hazmat.primitives.ciphers import Cipher, algorithms, modes
from cryptography.hazmat.backends import default_backend
import os

def encrypt_aes_gcm(plaintext: bytes, key: bytes, associated_data: bytes = None) -> tuple:
    nonce = os.urandom(12)
    cipher = Cipher(algorithms.AES(key), modes.GCM(nonce), backend=default_backend())
    encryptor = cipher.encryptor()
    if associated_data:
        encryptor.authenticate_additional_data(associated_data)
    ciphertext = encryptor.update(plaintext) + encryptor.finalize()
    return nonce, ciphertext, encryptor.tag

def decrypt_aes_gcm(ciphertext: bytes, key: bytes, nonce: bytes, tag: bytes, associated_data: bytes = None) -> bytes:
    cipher = Cipher(algorithms.AES(key), modes.GCM(nonce, tag), backend=default_backend())
    decryptor = cipher.decryptor()
    if associated_data:
        decryptor.authenticate_additional_data(associated_data)
    return decryptor.update(ciphertext) + decryptor.finalize()

Листинг 3 — Реализация RSA-2048 с OAEP

from cryptography.hazmat.primitives.asymmetric import rsa, padding
from cryptography.hazmat.primitives import hashes, serialization
from cryptography.hazmat.backends import default_backend

def generate_rsa_keypair(key_size: int = 2048):
    private_key = rsa.generate_private_key(
        public_exponent=65537,
        key_size=key_size,
        backend=default_backend()
    )
    public_key = private_key.public_key()
    return private_key, public_key

def encrypt_rsa(plaintext: bytes, public_key) -> bytes:
    return public_key.encrypt(
        plaintext,
        padding.OAEP(
            mgf=padding.MGF1(algorithm=hashes.SHA256()),
            algorithm=hashes.SHA256(),
            label=None
        )
    )

def decrypt_rsa(ciphertext: bytes, private_key) -> bytes:
    return private_key.decrypt(
        ciphertext,
        padding.OAEP(
            mgf=padding.MGF1(algorithm=hashes.SHA256()),
            algorithm=hashes.SHA256(),
            label=None
        )
    )

from cryptography.hazmat.primitives.asymmetric import ec

    return private_key, public_key

def ecdh_key_exchange(private_key, peer_public_key) -> bytes:
    shared_key = private_key.exchange(ec.ECDH(), peer_public_key)
    return shared_key

def generate_ed25519_keypair():
    private_key = ed25519.Ed25519PrivateKey.generate()
    public_key = private_key.public_key()
    return private_key, public_key

def sign_ed25519(message: bytes, private_key) -> bytes:

def verify_ed25519(message: bytes, signature: bytes, public_key) -> bool:
